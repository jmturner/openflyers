Pour récupérer (update) et/ou mettre à jour (commit) les fichiers CVS (Concurrent Version System) depuis SourceForge, un certain nombre de manipulations sont nécessaires.

Tout d'abord, il faut avoir un compte ouvert sur SourceForge. Cette opération est une démarche personnelle.

Ensuite, il faut être autorisé à l'accès des fichiers CVS d'OpenFlyers. Pour cela, il faut demander à l'équipe de développement OpenFlyers.

Une fois, ces primilaires effectués, il faut utiliser un outil pour créer une copie locale sur son ordinateur des fichiers CVS d'OpenFlyers.

Nous expliquons ci-dessous la démarche à suivre avec TortoiseCVS. Tout d'abord dans le cas d'un accès public (pas de compte ouvert ni d'autorisation requise) ensuite dans le cas d'un accès protégé.

Installation de TortoiseCVS pour récupérer les fichiers CVS en anomyne (sans possibilité de faire des modifications : accès public)

TortoiseCVS est un client CVS qui s'intègre à l'explorateur windows. Il est téléchargeable ici : http://www.tortoisecvs.org/

Après installation et reboot, faire dans un répertoire quelconque : clic droit puis "CVS emprunter".

Dans le 'CVSROOT', "chaine de connexion CVS", mettre: :pserver:anonymous@cvs.sourceforge.net:/cvsroot/openflyers



Installation de TortoiseCVS pour un accès complet aux fichiers CVS


Par rapport à l'image ci-dessus, il suffit de remplacer :
  • en protocole :pserver: par :ext: (secure shell)
  • en nom d'utilisateur le nom du compte sur sourceforge (sourceforge vérifiera alors que vous avez bien les droits d'accès protégé)
C'est tout.

Lors de chaque accès, il vous sera demandé un mot de passe. Ce mot de passe correspond au mot de passe de votre accès à sourceforge que vous avez choisi lorsque vous avez créé votre compte sur sourceforge.

Si vous voulez éviter d'entrer votre mot de passe à chaque commit/update depuis tortoiseCVS, vous pouvez créer un accès sécurisé à l'aide d'une clé public et d'une clé privée.Pour cela, il faut utiliser les outils  de la suite Putty.

Utilisation de la suite Putty pour générer un accès sécurisé à sourceforge

Vous avez besoin de :
  • putty
  • puttygen
  • pageant
Commencer par créer une double clé (public/privée à l'aide de puttygen)



Mettez en "parameters" SSH2 DSA, puis cliquez sur "generate" (il faut bouger la souris dans la zone de puttygen)



Une fois la clé générée, en commentaire mettez votre "username" de sourceforge suivi de "sourceforge.net".

Vous devez également saisir un mot de passe dans "key passphrase" et "confirm passphrase" qui vous sera demandé par la suite par Pageant.



Sauvez la clé privée sur votre disque dur. Utilisez comme extension ".ppk"



Puis rendez-vous sur sourceforge afin de fournir la clé public. Pour cela, il suffit d'aller sur la page d'édition des paramêtres de votre compte (account maintenance, ou account options), de cliquer sur "[Edit SSH Keys for Shell/CVS]" en bas de la page, et enfin de faire un copier/coller de la "public key for pasting into OpenSSH authorized_keys2 file" visible sur puttygen (il faut tout copier) et de le coller dans le formulaire de saisi de sourceforge. Cliquez sur update, laisser sécher selon le temps indiqué ici.

Vous pouvez quitter puttygen et lancer putty.



Configurez Host Name comme indiqué ci-dessus



Vérifiez sur la page 'SSH' que vous avez bien le protocole 2 de spécifié.



Et idem pour la page 'Auth'.

Enfin sauvez votre nouvelle configuration (session) sous le nom 'sourceforge' par exemple.

Si cela fonctionne correctement, vous devez avoir :

"Welcome to cvs1.sourceforge.net

This is a restricted Shell Account
You cannot execute anything here."

Paramétrage de pageant

Pour utiliser avec TortoiseCVS cette nouvelle configuration, il vous suffit de lancer pageant (avant TortoiseCVS). Cliquez sur son icone dans la barre des tâches, "ajouter une clé" en sélectionnant comme nom de fichier celui que vous avez précédemment créé avec PuttyGen. Il va vous demander un mot de passe, c'est celui que vous avez créé avec PuttyGen.

Vous pouvez maintenant utiliser TortoisePLink sans avoir à saisir de nouveau mot de passe.
